HTML (HyperText Markup Language)

HTML :

What it is: The standard markup language for creating the structure of web pages.

Purpose: Defines the layout of a web page using elements like headings, paragraphs, images, and links.

Key Features:

  • Tags (<h1>, <p>, <div>) to structure content.
  • Attributes (id, class, href) for adding metadata and styles.
  • Semantic elements (<header>, <footer>, <article>) for better SEO and accessibility.

CSS (Cascading Style Sheets)

CSS :

What it is: A style sheet language used for describing the presentation of a web page.

Purpose: Controls the layout, colors, fonts, and responsiveness of web pages.

Key Features:

  • Styling rules for elements (color: red;, font-size: 16px;).
  • Selectors (.class, #id, element) to target elements.
  • Media queries for responsive design.

JavaScript (JS)

JS :

What it is: A programming language for creating dynamic and interactive web content.

Purpose: Adds functionality like animations, form validation, and fetching data from APIs.

Key Features:

  • Event-driven programming (e.g., button clicks).
  • DOM manipulation for dynamic content.
  • ES6+ features like let, const, arrow functions, and async/await.

Bootstrap

Bootstrap :

What it is: A popular front-end framework for building responsive and mobile-first websites.

Purpose: Simplifies web development with pre-designed components and a responsive grid system.

Key Features:

  • Components like navbars, carousels, and modals.
  • A 12-column grid system for layout design.
  • Customizable via SASS variables.

MongoDB

MongoDB :

What it is: A NoSQL database that stores data in a flexible, JSON-like format.

Purpose: Handles large amounts of unstructured or semi-structured data.

Key Features:

  • Document-based storage (collections and documents).
  • Schema-less, allowing flexibility in data structure.
  • Querying with MongoDB Query Language (MQL).

Express.js

Express.js :

What it is: A lightweight, flexible Node.js framework for building web applications and APIs.

Purpose: Simplifies server-side development with routing and middleware.

Key Features:

  • Middleware for processing requests and responses.
  • RESTful API creation.
  • Integration with databases like MongoDB.

React.js

React.js :

What it is: A JavaScript library for building user interfaces, especially for single-page applications (SPAs).

Purpose: Manages the view layer and allows the creation of reusable UI components.

Key Features:

  • Component-based architecture.
  • Virtual DOM for efficient rendering.
  • Hooks like useState and useEffect.

Node.js

Node.js :

What it is: A JavaScript runtime built on Chrome's V8 engine.

Purpose: Allows JavaScript to run on the server side for building scalable network applications.

Key Features:

  • Asynchronous, event-driven architecture.
  • Built-in modules for file handling, networking, and more.
  • Package management using npm (Node Package Manager).
